# Архитектура Arachnel

## Слои

```
┌─────────────────────────────────────────┐
│  UI (QML) — библиотека, каталог, настройки │
├─────────────────────────────────────────┤
│  Core — библиотека, задачи, запуск, хранилище │
├─────────────────────────────────────────┤
│  Source plugins — Online-Fix, FreeTP, …     │
└─────────────────────────────────────────┘
```

### Core (ядро)

Ответственность ядра — то, что **одинаково для всех источников**:

- локальная библиотека (установленные игры, пути, версии);
- очередь задач (скачивание, установка, обновление);
- запуск процесса игры;
- настройки приложения (папки, тема, плагины);
- загрузка и жизненный цикл плагинов.

Ядро **не** знает, как распаковать конкретный архив Online-Fix или когда у FreeTP нужен отдельный фикс.

### Source plugin (плагин источника)

Один плагин = один тип источника (или семейство с общим API).

Плагин реализует свой пайплайн, например:

| Этап | Примеры различий между источниками |
|------|-----------------------------------|
| Каталог | API сайта, парсинг, локальный индекс |
| Метаданные | версия, размер, тип дистрибутива |
| Загрузка | прямая ссылка, несколько частей, зеркала |
| Установка | unzip portable, silent installer, копирование + патч |
| Обновление | diff, полная перекачка, замена папки |
| Запуск | `exe` в корне, подпапка `Binaries`, аргументы |

### UI (QML)

- оболочка (`qml/app/`);
- экраны библиотеки и каталога (будущее);
- настройки (`qml/settings/`);
- тема (`qml/theme/`).

UI вызывает Core; Core делегирует операции с каталогом/установкой активному плагину источника.

## Контракт плагина источника (черновик)

Минимальный набор возможностей, который ядро ожидает от плагина:

```text
id              — уникальный идентификатор ("online-fix", "freetp", …)
name            — отображаемое имя
capabilities    — search | install | update | launch_hints

search(query)   → список игр в каталоге источника
metadata(id)    → версия, размер, тип установки, ссылки
install(job)    → прогресс: download → prepare → finalize
detectUpdate(local) → есть ли новая версия
launchInfo(local)   → executable, cwd, args (или подсказки для ядра)
```

Тип установки (`installKind`) — чтобы ядро показывало правильный UX, но **логику выполняет плагин**:

- `portable_archive` — распаковка;
- `installer` — запуск установщика / тихая установка;
- `bundled_fix` — уже готовая сборка;
- `fix_download` — основная игра + отдельная загрузка фикса.

## Примеры плагинов

### online-fix

- дистрибутив: чаще portable-архив;
- установка: скачать → распаковать в `{library}/{appId}` → записать манифест;
- обновление: сравнить версию с каталогом → перекачать/заменить.

### freetp

- дистрибутив: **зависит от страницы игры**;
- установка: ветвление внутри плагина (installer / portable / fix overlay);
- обновление: по правилам конкретной раздачи.

## Структура репозитория (целевая)

```
src/
  core/           # библиотека, задачи, plugin host
  plugins/        # встроенные или подключаемые плагины (C++/Python — TBD)
qml/
  app/            # LibraryPage, CatalogPage, …
plugins/
  online-fix/     # манифест + реализация источника
  freetp/
docs/
```

Сейчас реализованы только UI-скелет и тема; Core и плагины — следующие этапы.

## Отличие от Hydra в одном предложении

Hydra оптимизирован под **один** способ наполнить библиотеку; Arachnel оптимизирован под **много способов**, каждый из которых инкапсулирован в плагине источника.
