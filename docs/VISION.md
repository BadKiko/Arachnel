# Arachnel — видение проекта

## Что это

**Arachnel** — десктоп-лаунчер игр в духе [Hydra Launcher](https://github.com/hydralauncher/hydra), но с другой реализацией и другим фокусом.

| | Hydra | Arachnel |
|---|--------|----------|
| Основной сценарий | Универсальная библиотека + один способ загрузки/запуска | **Online-fix** и смежные источники |
| Источники | По сути одна система (торренты, debrid, репаки) | **Плагинная система источников** |
| Установка | Относительно единый пайплайн | Зависит от источника: portable, installer, встроенный фикс, отдельный патч |

## Зачем плагины источников

У разных площадок разная логика доставки игры. Один универсальный установщик это не покрывает.

**Online-Fix** — чаще portable-архивы: скачать → распаковать → запустить.

**FreeTP** — смешанные случаи:
- обычный `.exe` установщик;
- portable без установки;
- фикс уже внутри сборки;
- фикс нужно докачать и применить отдельно.

Чтобы не раздувать ядро ветвлениями `if (source == …)`, каждый источник — **отдельный плагин** со своим пайплайном: поиск, загрузка, установка, обновление, подготовка к запуску.

## Цели лаунчера

1. **Библиотека игр** — что установлено, откуда, какая версия.
2. **Автоустановка и обновления** — через API/логику плагина источника.
3. **Запуск** — единая точка «Играть», параметры и рабочая папка задаёт плагин или ядро по метаданным установки.
4. **Расширяемость** — новый источник = новый плагин, без переписывания ядра.

## Не цели (пока)

- Копировать Hydra один в один (торренты, debrid, их UI).
- Один жёсткий формат дистрибутива для всех источников.

## Принцип разработки

Ядро знает **контракт** плагина и общий UX. Плагин знает **как** работать с конкретным источником.

Подробнее о слоях и контракте: [ARCHITECTURE.md](ARCHITECTURE.md).
